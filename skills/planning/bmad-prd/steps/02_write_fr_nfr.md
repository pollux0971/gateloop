# Step 2 — Write Functional & Non-Functional Requirements (itemized)

Itemize the requirements so each is independently testable.

- **Functional Requirements** — one per line, prefixed `FR-1`, `FR-2`, … Each
  must be *testable*: state an observable behaviour ("the system shall …"),
  ideally with a Given/When/Then or a measurable condition. No vague verbs.
- **Non-Functional Requirements** — one per line, prefixed `NFR-1`, `NFR-2`, …
  (performance, security, accessibility, etc.), each with a measurable target.

Leave no `TBD` / `TODO` / `<placeholder>` behind — the checklist rejects them.
Finish by writing the Success criteria (measurable outcomes).
