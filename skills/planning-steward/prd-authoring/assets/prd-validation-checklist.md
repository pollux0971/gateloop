# PRD validation checklist
- [ ] Vision stands alone (no external context needed)
- [ ] Every feature has ≥1 FR; every FR has a stable id
- [ ] Every FR references the UJ it realises
- [ ] Glossary covers every domain noun; no synonyms used elsewhere
- [ ] NFRs cover performance, security, reliability as applicable
- [ ] Success metrics are measurable
- [ ] Assumptions tagged inline and indexed
- [ ] No architecture/implementation detail leaked into the PRD
